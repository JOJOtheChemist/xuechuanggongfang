# 高考自然语言提问返回样例与检索结果

## 文档目的
这份文档专门记录：

- 用户用自然语言提问时，`yunnan-gaokao-volunteer-consultant-v3` 这一套链路会怎么触发 skill
- 自然语言大概会被清洗成什么关键词，再传给高考检索工具
- 当前工具真实返回了什么学校/详情结果
- 哪些地方已经符合预期，哪些地方还存在明显跑偏

这份文档适合后面排查：

- skill 有没有触发错
- 关键词有没有抽错
- tool 是不是把“整句问法”错传进去了
- 学校详情和冲稳保解释是不是对家长学生足够友好

## 本次环境限制
- 当前本机默认主后端仍指向 `http://localhost:8000`，本地未启动会直接失败
- 切到 `https://xuechuang.xyz` 后，已登录链路所需 token 当前失效，返回 `401 ACCESS_TOKEN_INVALID`
- 所以这次大部分高考检索结果，实际走的是游客 `preview` 链路
- 这意味着：
- 能验证“skill 是否触发”“关键词怎么提取”“工具会返回什么学校”
- 但不能完整验证“已登录正式链路下的 riskBucket / strategyLabel / 冲稳保真实分档”

## 本次核心结论
- `zhangxuefeng-perspective` 能正常按关键词 `张雪峰` 触发
- 当前自然语言不会再整句原样塞进 `query`，已经会先做一轮关键词清洗
- 但清洗结果还有提升空间，尤其是：
- 文科问法里会残留类似“什么”“喜欢新闻”这种不够干净的词
- 学校详情在游客链路下仍可能出现明显误命中
- 当前游客链路返回的学校结果偏“学校池预览”，不等于真正的冲稳保正式推荐结果
- `interpret_admission_risk` 这类解释工具本身输出已经比较像家长能听懂的话

## 样例 1
用户问题：

```text
云南文科450分，喜欢新闻，推荐什么学校
```

实际情况：

- skill 触发：否
- skill 原因：无
- 清洗后的 `query`：`什么 喜欢新闻`
- 抽取的 `majorKeyword`：`新闻`
- 实际调用工具：`search_yunnan_admission`
- 工具模式：`preview`

实际返回摘要：

- 返回学校数量：`1`
- 首个学校：`云南大学`
- 学校层次：`本科`
- 风险档位：空

我看到的问题：

- 这条问法本质上是在问“文科 450 分 + 新闻方向，能看什么学校”
- 目前系统确实抓到了 `新闻` 这个专业方向
- 但 `query` 里还残留了 `什么 喜欢新闻`，说明自然语言瘦身还不够干净
- 这类文科问题，后面最好让 Agent 不只是按专业词搜学校，还要更主动解释：
- 学校所在城市
- 学校层次
- 是否适合文科生的院校氛围

## 样例 3
用户问题：

```text
用张雪峰的视角帮我看下，云南理科560分学金融到底值不值得报
```

实际情况：

- skill 触发：是
- 触发 skill：`zhangxuefeng-perspective`
- skill 原因：`zhangxuefeng-perspective:keyword:张雪峰`
- 清洗后的 `query`：`金融`
- 抽取的 `majorKeyword`：`金融`
- 实际调用工具：`search_yunnan_admission`
- 工具模式：`preview`

实际返回摘要：

- 返回学校数量：`1`
- 首个学校：`云南大学`
- 学校层次：`本科`
- 风险档位：空

我看到的问题：

- skill 触发这一层是通的
- 检索也已经不再把整句“张雪峰视角 + 560 分 + 值不值得报”原样传给工具
- 这说明“style 问法把搜索拖死”的问题已经比之前好很多
- 但当前游客链路下，结果仍然偏“命中学校池”，还不是一个真正能回答“金融值不值得报”的完整答复
- 真正上线时，Agent 还需要在工具结果之上补解释：
- 金融在云南本地/省外的选择逻辑
- 学校层次差异
- 就业导向
- 张雪峰风格下的“值不值得”判断

## 样例 4
用户问题：

```text
昆明医科大学口腔医学怎么样
```

实际情况：

- skill 触发：否
- 清洗后的 `query`：`口腔医学 昆明医科大学`
- 抽取的学校名：`昆明医科大学`
- 实际调用工具：`search_yunnan_admission_school_detail`

实际返回摘要：

- `institution_query`：`昆明医科大学`
- 实际命中学校：`昆明冶金职业大学`
- 学校层次：`专科`
- 专业数量：`0`

我看到的问题：

- 这条是本次最重要的一个“跑偏样例”
- 用户明明问的是 `昆明医科大学`
- 但游客详情链路最后命中成了 `昆明冶金职业大学`
- 这说明当前详情查询虽然已经能抽出学校名，但在预览检索阶段仍然可能被模糊匹配带偏
- 这个问题必须单独盯：
- 如果正式链路也会这样，就会直接影响家长信任
- 这类“学校名明确”的问题，后面最好增加更强的精确匹配优先级

- 新闻学是不是该校重点方向

## 样例 6
工具级风险解释样例

输入：

```text
school_name=昆明医科大学
risk_bucket=stable
score=500
reference_score=495
city=昆明
school_level=本科
strategy_label=稳妥
```

实际返回要点：

- 风险标签：`较稳/稳妥`
- 解释文案：

```text
对于 昆明医科大学，当前属于“较稳/稳妥”档。说明这所学校和当前分数匹配度比较均衡，属于比较适合重点考虑的稳妥选择。
```

- 导师口播风格摘要：

```text
它被放在“较稳/稳妥”这一档，核心意思不是简单说能不能上，而是说它和你当前分数的匹配风险大概在这个区间。
```

这个结果说明：

- `interpret_admission_risk` 这层已经比较适合给家长学生直接听
- 后面真正要补强的不是“解释语气”，而是前面学校检索和正式 riskBucket 的真实性

## 当前最值得继续改的点
- 继续优化自然语言清洗
- 尤其是去掉 `什么`、`喜欢新闻` 这类残留问句词
- 对“明确学校名”的详情查询加更强的精确匹配
- 避免 `昆明医科大学` 被带偏到 `昆明冶金职业大学`
- 文科问题要更主动解释学校地理位置和院校层次
- 理科问题要更主动解释专业导向和分数匹配度
- 真正的冲稳保验证，必须补一轮“有效登录 token + 正式链路” smoke
