## 我测试了什么

我按你说的把 demo 里两种模式都测了一遍：

1. 搜索版：`SearchType = web`
2. 总结版：`SearchType = web_summary`

另外我也顺手复测了火山方舟官方 `Responses API + web_search`，用来确认你说的“刚开通 websearch”到底是开通了哪一套。

## 测试 1：demo 搜索版

请求地址：

- `https://open.feedcoopapi.com/search_api/web_search`

请求模式：

- `SearchType: web`

请求结果：失败

返回结果核心信息：

```json
{
  "ResponseMetadata": {
    "Error": {
      "Code": "invalid_api_key",
      "Message": "invalid api key"
    }
  }
}
```

结论：

- 这个 demo 所指向的 `feedcoopapi` 接口没有接受当前我用于测试的 key
- 也就是说，这一套“apikey 访问方式”需要它自己的独立 API key，不能直接拿当前火山方舟的 `ark-...` key 代替

## 测试 2：demo 总结版

请求地址：

- `https://open.feedcoopapi.com/search_api/web_search`

请求模式：

- `SearchType: web_summary`

请求结果：失败

返回结果核心信息：

```json
{
  "ResponseMetadata": {
    "Error": {
      "Code": "invalid_api_key",
      "Message": "invalid api key"
    }
  }
}
```

结论：

- 总结版和搜索版一样
- 当前缺的不是请求格式，而是这套 `feedcoopapi` 服务对应的正确 API key

## 测试 3：火山方舟官方 web_search

为了确认“你刚开通的 websearch”是不是火山方舟官方能力，我又测了：

- `POST https://ark.cn-beijing.volces.com/api/v3/responses`
- 模型：`doubao-seed-2-0-lite-260428`
- 工具：`web_search`

请求结果：失败

返回结果核心信息：

```json
{
  "error": {
    "code": "ToolNotOpen",
    "type": "NotFound"
  }
}
```

结论：

- 截至 2026-05-20，这把方舟 key 在官方 `Responses API` 上仍然被判定为 `web search` 工具未开通
- 所以你刚刚开通的权限，大概率不是方舟这条链路，或者还没有完成生效

## 最终判断

现在这件事要分成两套系统看：

1. `feedcoopapi` demo 这套
- 搜索版失败
- 总结版失败
- 原因一致：`invalid_api_key`
- 说明需要一把专门给 `https://open.feedcoopapi.com/search_api/web_search` 用的 API key

2. 火山方舟官方这套
- 模型 `doubao-seed-2-0-lite-260428` 聊天已经可用
- 但 `web_search` 工具仍返回 `ToolNotOpen`
- 说明官方联网搜索权限截至 2026-05-20 仍未真正打通
