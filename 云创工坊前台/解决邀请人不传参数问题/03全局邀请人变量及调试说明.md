# 三、全局邀请人变量及调试说明 🔍

为了方便在各个页面随时监控用户的登录状态及邀请绑定关系，我们在 `main.js` 中实现了**全局变量自动注入**，并在关键页面底部增加了**统一调试栏**。

## 1. 四大全局变量定义

所有页面（`.vue`）均可直接使用以下变量，无需额外声明：

| 变量名 | 含义 | 数据来源优先级 | 说明 |
| :--- | :--- | :--- | :--- |
| **`global_uid`** | 当前用户UID | 1. `getCurrentUserInfo().uid`<br>2. `storage.userId` | 解决了 uni-id 在某些情况下无法识别自定义登录态的问题（僵尸号修复）。 |
| **`global_lifetime_inviter`** | **终身绑定邀请人** | 1. `userInfo.inviter_uid`<br>2. `partner_info.inviter_id` | **不可变**。用户注册或首次绑定后永久生效，用于返佣结算。 |
| **`global_team_inviter`** | **当前团队邀请人** | `storage.pending_team_invite` | **临时变量**。扫码入团时携带，随支付订单核销后失效。用于标识这次是为了加入谁的团队。 |
| **`global_business_inviter`** | **当前业务邀请人** | `storage.pending_business_invite` | **临时变量**。扫码报名学习板块时携带，随业务报名成功后失效。 |

---

## 2. 核心实现原理 (Global Mixin)

在 `main.js` 中通过 `Vue.mixin` 实现了全局混入。每当页面 `onShow`（显示）时，都会自动刷新这些变量：

```javascript
// main.js 部分代码参考
onShow() {
    const userInfo = uniCloud.getCurrentUserInfo()
    
    // 1. 修复 UID 显示
    this.global_uid = userInfo.uid || uni.getStorageSync('userId')

    // 2. 获取终身邀请人 (优先查标准字段，再查项目自定义字段)
    const storageInfo = uni.getStorageSync('userInfo') || {}
    this.global_lifetime_inviter = userInfo.inviter_uid || 
                                   storageInfo.inviter_uid || 
                                   (storageInfo.partner_info && storageInfo.partner_info.inviter_id) || 
                                   '未绑定'

    // 3. 获取临时邀请人缓存
    const pendingTeam = uni.getStorageSync('pending_team_invite')
    this.global_team_inviter = pendingTeam?.inviter || '无'
    
    // ... 业务邀请人同理
}
```

## 3. 统一调试栏 (Unified Debug Footer)

我们在以下关键页面的最底部增加了半透明的调试信息栏，用于实时观察变量变化：

1.  **登录页** (`pages/auth/login/index`)
2.  **首页/仪表盘** (`pages/dashboard/index`)
3.  **浏览团队列表** (`pages/extra/team-browser`)
4.  **加入团队确认页** (`pages/extra/join-team-confirm`)

**显示格式：**
> UID: 6643... | 终身: 1234... | 团队: 8888... | 业务: 无

### 💡 如何验证？
*   **未登录时**：UID 应显示“未登录”或本地缓存的旧 ID。
*   **扫码后**：
    *   扫团队码 -> 观察“**团队**”字段是否有值。
    *   扫业务码 -> 观察“**业务**”字段是否有值.
*   **登录/支付后**：
    *   如果是新用户，注册成功后，“**终身**”字段应立即更新为邀请人 ID。
    *   支付完成后，相关临时字段（团队/业务）通常会在逻辑处理完后被清除（变为“无”），这是正常现象。
