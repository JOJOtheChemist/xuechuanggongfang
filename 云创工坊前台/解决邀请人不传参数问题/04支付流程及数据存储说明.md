# 四、支付流程及数据存储说明 💰

## 支付成功的标志

### 主要标志
在 `payment_orders` 数据表中，订单的 `status` 字段变化：
- `created` → `paid` ✅ **这就是支付成功的标志！**

### 支付确认方式
支付成功有两种确认途径：
1. **客户端确认** - 用户支付完成后，前端调用 `confirmPayment()`
2. **服务器回调** - 微信支付服务器调用 `paymentNotify()` (更可靠)

## 支付流水存储位置

### 1. 核心数据表：`payment_orders` (支付订单表)

这是**所有支付流水的主表**，存储所有支付订单记录。

#### 关键字段说明

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `order_no` | String | 订单号（唯一标识） |
| `user_id` | String | 支付用户的 UID |
| `business_id` | String | 业务ID（如 `team_join_xxx`） |
| `business_name` | String | 业务名称（如"加入XX团队"） |
| `amount` | Number | 支付金额（元） |
| `total_fee` | Number | 支付金额（分） |
| **`status`** | **String** | **订单状态** ⭐ |
| `pay_time` | Number | 支付完成时间戳 |
| `create_date` | Number | 订单创建时间戳 |
| `extra_data` | Object | 额外业务数据 |
| `signup_id` | String | 关联的报名记录ID（可选） |
| `join_team_id` | String | 关联的团队ID（入团订单） |

#### 订单状态值 (`status`)

| 状态值 | 含义 | 说明 |
|--------|------|------|
| `created` | 已创建 | 订单已生成，等待支付 |
| **`paid`** | **已支付** | **支付成功** ✅ |
| `failed` | 支付失败 | 支付过程出错 |
| `cancelled` | 已取消 | 用户取消支付 |

---

### 2. 辅助数据表

#### `team_join_logs` - 入团日志表
记录所有入团操作的详细日志：
- `team_id` - 加入的团队ID
- `inviter_id` - 邀请人ID
- `new_user_id` - 新成员ID
- `order_no` - 关联的支付订单号
- `status` - 入团状态
- `amount` - 支付金额
- `create_date` - 入团时间

#### `points_logs` - 积分流水表
记录积分变动（包括推荐人奖励）：
- `user_id` - 用户ID
- `change` - 积分变化量（正数为增加）
- `balance_after` - 变动后余额
- `reason` - 变动原因（如 `recommend_reward`）
- `ref_id` - 关联订单ID
- `remark` - 备注说明

#### `payment_withdraws` - 提现记录表
记录用户余额提现申请：
- `user_id` - 用户ID
- `amount` - 提现金额（元）
- `status` - 提现状态（`processing`/`success`/`failed`）
- `order_no` - 提现订单号

## 相关云对象

### 1. `payment-service` - 核心支付服务

**文件位置：** `uniCloud-aliyun/cloudfunctions/payment-service/index.obj.js`

#### 主要方法

| 方法名 | 功能 | 说明 |
|--------|------|------|
| `createOrder()` | 创建支付订单 | 调用微信统一下单接口，生成支付参数 |
| `confirmPayment()` | 确认支付 | 客户端支付成功后调用，更新订单状态为 `paid` |
| `paymentNotify()` | 支付回调 | 微信服务器回调，服务端确认支付成功 |
| `queryOrder()` | 查询订单 | 查询订单状态和详情 |
| `withdraw()` | 申请提现 | 用户余额提现申请 |
| `getWithdrawRecords()` | 提现记录 | 获取用户提现历史 |

---

### 2. `team-join-service` - 入团支付专用服务

**文件位置：** `uniCloud-aliyun/cloudfunctions/team-join-service/index.obj.js`

#### 主要方法

| 方法名 | 功能 | 说明 |
|--------|------|------|
| `createJoinOrder()` | 创建入团订单 | 固定金额 0.1 元，调用 `payment-service` |
| `confirmJoin()` | 确认入团 | 验证支付状态，执行入团逻辑（事务） |

#### 入团流程说明

```
1. 用户点击"加入团队"
   ↓
2. createJoinOrder() 
   - 验证团队存在
   - 创建支付订单 (status: 'created')
   - 调用微信统一下单
   ↓
3. 用户完成微信支付
   ↓
4. confirmJoin()
   - 确认订单状态为 'paid'
   - 开启数据库事务
   - 更新用户 team_info
   - 更新团队 member_count
   - 写入 team_join_logs
   - 提交事务
   ↓
5. 入团成功 ✅
```

## 常见问题排查

### Q1: 如何查看某个用户的支付记录？

```javascript
// 在 uniCloud 控制台 - 云数据库中查询
db.collection('payment_orders')
  .where({ user_id: '用户UID' })
  .orderBy('create_date', 'desc')
  .get()
```

### Q2: 如何确认支付是否成功？

查看订单的 `status` 字段：
- `status === 'paid'` → 支付成功 ✅
- `status === 'created'` → 待支付
- 同时检查 `pay_time` 字段是否有值

### Q3: 入团支付后没有加入团队？

检查以下几点：
1. `payment_orders` 表中订单 `status` 是否为 `paid`
2. `team_join_logs` 表中是否有对应记录
3. 用户表 `uni-id-users` 中 `team_info.team_id` 是否已更新
4. 查看云函数日志，确认 `confirmJoin()` 是否执行成功

### Q4: 事务错误 "transaction not support action [find]"

**原因：** 在事务中使用了 `.where()` 查询

**解决：** 改用 `.doc(id).get()` 直接通过 ID 获取文档

```javascript
// ❌ 错误写法
const userRes = await transaction.collection('uni-id-users')
  .where({ _id: uid })
  .get()

// ✅ 正确写法
const userRes = await transaction.collection('uni-id-users')
  .doc(uid)
  .get()
const user = userRes.data  // 注意：.doc() 返回单个对象，不是数组
```

## 数据查询示例

### 查询今日支付成功的订单

```javascript
const today = new Date()
today.setHours(0, 0, 0, 0)
const todayStart = today.getTime()

const res = await db.collection('payment_orders')
  .where({
    status: 'paid',
    pay_time: db.command.gte(todayStart)
  })
  .orderBy('pay_time', 'desc')
  .get()
```

### 查询某个团队的入团记录

```javascript
const res = await db.collection('team_join_logs')
  .where({ team_id: '团队ID' })
  .orderBy('create_date', 'desc')
  .get()
```

### 查询推荐人获得的积分奖励

```javascript
const res = await db.collection('points_logs')
  .where({
    user_id: '推荐人UID',
    reason: 'recommend_reward'
  })
  .orderBy('create_date', 'desc')
  .get()
```
